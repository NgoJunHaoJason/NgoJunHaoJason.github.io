Algorithm Analysis

What is it?

We analyse algorithms to quantify their resource consumption.

To be more specific, we do asymptotic algorithm analysis: study of the resources used by an algorithm when the problem becomes larger and larger without bound.

To do so, we need comparison criteria or measures.

In this course, we measure algorithms' efficiency in terms of time and space taken.

So how do we do it?

A naive approach is to do empirical comparison, where you actually run the programs and measure the time taken to run them.

However, there are several problems with doing an empirical comparison:

  • It is difficult to do it 'fairly'.
    • When you run a program on a machine, other factors come into play.
    • For example, different CPUs may have different clock speed and take different amount of time to run programs.
    • In addition, otther programs that are running in the background would require the CPU and contribute to the time taken.
    • Also, during the first run of a program, some values are accessed for the first time and for subsequent runs, they are readily available in the cache, resulting in inconsistent time taken.
  • Execution time is not the right measure of time efficiency.
    • In order to run the program, you have to enter data of known, finite problem size. However, for asymptotic analysis, there is supposedly no upper bound on problem - it can get as large as possible.
    • Points regarding fairness of empirical comparisons applies here as well.
  • It is time consuming to perform empirical comparisons.
      The problem size is typically huge, which means a lot of time is taken to process the data.

The smarter way of measuring efficiency is to derive the time and space complexity of the algorithm.

To measure time complexity (amount of time used), we:

  1. Count the number of primitive operations.
  2. Determine the major parameters that affect the problem (i.e. problem size).
  3. Derive an equation that relates the parameters to the number of primitive operations that the algorithm does.
This is related to algorithmic aspects.

To measure space complexity (amount of space used), we:

  1. Count the number of storage units required. (e.g. memory used - 1 int → 4 bytes for instance)
  2. Determine the major parameters that affect the problem (problem size).
  3. Derive an equation that relates the parameters to the number of storage units that the algorithm uses.
This is related to data structures.

Some Terminologies

problem size

  • The problem size depends on the problem being studied (e.g. the number of items to be sorted).
  • Usually, the number of primitive operations done by an algorithm is a function of the problem size.

primitive operation

  • A basic step that can be performed in constant time.
  • Examples:
    • Assignment (e.g. x = 1)
    • Arithmetic operations (+, -, *, /)
    • Comparisons (==, !=, <, >, <=, >=)

programming constructs

  • A composite collection of basic steps.
  • Examples:
    • loops (for, while)
    • branching (if-else, switch-case)
    • functions (recursive vs non-recursive)
