Introduction

An algorithm is a clearly specified set of simple instructions to be followed to solve a problem or compute a function.

A good algorithm possesses the following properties:

  1. It must be correct - it always returns the correct results.
  2. It must be composed of a series of concrete steps. There can be no ambiguity as to which step will be performed next.
  3. It must be composed of a finite number of steps - it must terminate.

At the heart of computer program design, there are two (sometimes conflicting) goals:

  1. To design an algorithm that is easy to understand, code and debug, which can be achieved through software engineering.
  2. To design an algorithm that makes efficient use of the computer's resources, which can be achieved through data structures and algorithm analysis

This course focuses on the most efficient algorithms - ones that uses the least amount of resources.
