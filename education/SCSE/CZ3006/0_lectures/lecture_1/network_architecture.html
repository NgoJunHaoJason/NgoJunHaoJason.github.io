Network Architecture

motivations for layered network architecture

Networks are complex with many pieces: hosts, routers, links, applications, protocols, hardware, software.

As such, there is a need to decompose the network into several layers.

An explicit structure allows identification of different pieces and relationships between different pieces.

Modularisation of the network eases maintenance and updating.

  • changes of a layer's service is transparent to the rest of the system
  • change in network technology does not affect the rest of the system

benefits of layered architecture

  • simplicity - Network is easy to design once layers and their interactions are defined clearly.
  • flexibity - It is easy to modify and develop networks by modifying layers separately.
  • incremental changes - It is easy to add new layers and easy to add new functions to a layer.

disadvantages of layered architecture

  • need to maintain different layers
  • incur additional overhead

layered network architecture

The network architecture defines specifications of layers and protocols.

The network is organised as a stack of layers, with each layer offering its services to the layer above it and passing data and control to the layer below, using a well-defined interface.

The protocols form a set of rules which govern communication between network entities. They define format, order of messages sent and received among network entities, and actions taken upon message transmission and receipt.

The Open System Interconnection 7-layer model is a functional decomposition of the network. Only peer (adjacent) layers communicate with each other, with each layer adding / removing headers.

OSI in action

A message begins at the top (application layer) and moves down the OSI layers to the bottom (physical) layer.

As the message descends, each successive layer adds a header to it.

The header is header-specific information that explains what functions the layer carries out.

At the receiving end, headers are stripped from the message as it travels up the OSI layers.

OSI protocols

Layer n in one machine interacts with layer n in another machine to provide a service to layer n + 1.

The entities comprising the corresponding layers on different machines are called peer processes.

The machines use a set of rules and conventions called the layer n protocol.

Layer n peer processes communicate by exchanging Protocol Data Units.

OSI services

Communication between peer processes is virtual and indirect.

Layer n + 1 transfers information by invoking the services provided by layer n.

Services are available at Service Access Points.

Each layer passes data and control information to the layer below it until the physical layer is reached and transfer occurs.

The data passed to the layer below it is called Service Data Unit. Service Data Units are encapsulated in Protocol Data Units.

The Transmission Control Protocol / Internet Protocol model has the same behaviour as the OSI model.

The implementation of network architecture as a 5-layers model is a compromise between OSI model's 7 layers (easy to maintain, but too much overhead) and 1 layer (too complex to maintain).
