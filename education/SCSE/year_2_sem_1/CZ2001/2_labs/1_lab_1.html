Example Class 1 (Week 4 â€“ Week 5)

Each group will have TWO students to present for ONE of the following questions.

Perform complexity analysis for the algorithm based on the following steps:

  1. find the recurrence equation
  2. solve the recurrent equation
  3. conclude on the complexity class for the algorithm by a proof
The presentation should include how the recurrence equation is derived, how it is solved and prove the asymtotic complexity.

For this session, you should be able to prepare for your presentation completely within the first 30-40 minutes of your class.

1. Analyze the number of disk moves in terms of n (to be done by group 1)

At n = 0 and n = 1, the number of disk moves are 0 and 1 respectively, therefore:
W(0) = 0, W(1) = 1

At n > 1, the function calls itself twice, with n - 1, and performs 1 disk move, so:
W(n) = 2 * W(n-1) + 1

Solve the recurrence relation:
W(n) = 2 * W(n-1) + 1
   = 2 * [2 * W(n-1) + 1] + 1
   = 2 * 2 * W(n-2) + 2 + 1
   = 2 * 2 * [2 * W(n-3) + 1] + 2 + 1
   = 2 * 2 * 2 * W(n-3) + 4 + 2 + 1
   = 23 * W(n-3) + 22 + 21 + 20
   = ...
   = 2i * W(n-i) + 2i-1 + ... + 22 + 21 + 20
   = ...
   = 2n * W(n-n) + 2n-1 + ... + 22 + 21 + 20
   = 2n * W(0) + n-1∑i=0 2i
   = 2n * 0 + 20 * (2n - 1/2 - 1)
   = 2n - 1

Therefore, the number of disk moves is 2n - 1.

reference: geometric series

2. Analyze the number of multiplications in terms of n. (to be done by group 2)

At n = 1, there is no multiplication within 1 function call, so:
W(1) = 0

At n > 1, there are (n - 1) multiplications within 1 function call due to the for-loop.

The function then calls itself with n - 1 as input, such that:
W(n) = (n - 1) + (n - 2) + ... + 1 + 0
   = n * (n - 1)/2

Therefore, there are n/2 * (n - 1) multiplications.

3. Analyze the number of additions in terms of n. Assume that n = 2k. (to be done by group 3)

At n = 1, there is 0 addition, so:
W(1) = 0

At n > 1, there is 1 addition in a function call, along with a call to itself with n/2, resulting in recurrence of the form:
W(n) = W(n/2) + 1

which is equivalent to:
W(2k) = W(2k/2) + 1
   = W(2k-1) + 1
   = [W(2k-2) + 1] + 1
   = W(2k-2) + 2
   = [W(2k-3) + 1] + 2
   = W(2k-3) + 3
   = ...
   = W(2k-i) + i
   = ...
   = W(2k-k) + k
   = W(20) + log2n
   = W(1) + log2n
   = 0 + log2n
   = log2n

Therefore, there are log2n
additions.

4. Analyze the number of comparisons between array elements (between min1 and min2 throughout the recursive calls) in terms of n, the size of the array A. Assume that n = 2k. (to be done by group 4)

When there is only 1 (or less) element in the array, there is 0 comparison, so:
W(1) = 0

At n > 1, the function calls itself twice with n/2 and 1 element comparison (not index comparison),so
W(n) = 2 * W(n/2) + 1

which is equivalent to:
W(2k) = 2 * W(2k/2) + 1
   = 2 * W(2k-1) + 1
   = 2 * [2 * W(2k-2) + 1] + 1
   = 22 * W(2k-2) + 2 + 1
   = ...
   = 2i * W(2k-i) + 2i-1 + 21 + 20
   = ...
   = 2k * W(2k-k) + 2k-1 + 21 + 20
   = 2k * W(20) + k-1∑i=0 2i
   = 2k * W(1) + 1 * (2k - 1 / 2 - 1)
   = 2k * 0 + 2k - 1
   = 0 + n - 1

Therefore, there (n - 1) comparisons between array elements.

5. For the function minimum() in Question 4, analyze the number of divisions in terms of n. Assume that n = 2k. (to be done by group 5)

Flow control in minimum() is such that whenever there is an element comparison, there will be a division.

As such, there are (n - 1) divisions.

6. Analyze the total number of additions and subtractions in terms of n. (to be done by group 6)

When n = 1, there is no addition and subtraction, so:
W(1) = 0

When n > 1, the function calls itself with (n - 1), and perform 1 addition and 1 subtraction, such that:
W(n) = W(n-1) + 2
   = [W(n - 2) + 2] + 2
   = W(n - 2) + 2 * 2
   = ...
   = W(n - i) + 2 * i
   = ...
   = W(n - (n - 1)) + 2 * (n - 1)
   = W(1) + 2 * (n - 1)
   = 0 + 2 * (n - 1)

Hence, there are (2n - 2) additions and subtractions in total.
